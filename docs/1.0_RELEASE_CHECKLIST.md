# Tether 1.0 Release Checklist

This supplements `RELEASE.md` for the 1.0 readiness pass. PR #113 already covered runtime hardening around transport startup, TLS defaults, redaction, corrupt data recovery, navigation guards, status detector cleanup, and ESLint restoration. Do not redo that work unless a merge conflict or release-blocking regression requires it.

## Baseline

- Start release hardening branches from current `github/main`.
- Keep the main checkout clean and use a dedicated worktree for release-readiness changes.
- Confirm `package.json` version, `CHANGELOG.md`, and release notes are intentional before cutting a 1.0 tag.
- Keep Helm stabilization out of scope unless `npm --prefix mcp-servers/tether-helm run build` or packaging fails.

## Audit Gates

Run these before the 1.0 release PR is merged:

```bash
npm audit
npm audit --omit=dev
npm --prefix mcp-servers/tether-helm audit
```

Expected posture after the 1.0 build-hardening pass:

- Runtime dependency audit: zero findings.
- Helm MCP audit: zero findings.
- Full root audit: zero findings.

Do not use `npm audit fix --force` for this tree without reviewing the resulting Forge/Vite/Electron major changes. The current patched path is:

- `vite` on a patched 6.x line to clear the Vite/esbuild dev-server advisories.
- `tar` overridden to a patched 7.x release to clear the node-tar advisory path while Forge still depends on `@electron/rebuild@3`.
- `tmp` overridden to a patched 0.2.x release for the Inquirer editor chain.
- `make-fetch-happen` overridden to 12.0.0 to remove the low-severity Forge build-chain path through `@electron/node-gyp` -> `http-proxy-agent` -> `@tootallnate/once` while staying on stable Forge 7.x.

Revisit the `make-fetch-happen` override when Electron Forge releases a stable dependency path that no longer pulls the old `@electron/rebuild`/`@electron/node-gyp` stack. Do not move to Forge 8 alpha or `@electron/rebuild@4` only for audit noise without package/make smoke, because those paths currently require a Node 22.12+ build toolchain.

## Build And Smoke

Run the normal verification set:

```bash
npx tsc --noEmit
npm test
npm run lint
npm --prefix mcp-servers/tether-helm run build
npm run package
npm run make
```

Package/make smoke criteria:

- `npm run package` completes with ASAR enabled.
- `npm run make` produces the Squirrel setup executable and portable zip in `out/make`.
- A packaged app can launch, open the main window, and create at least one local session.
- Native modules remain available from `app.asar.unpacked`.
- `resources/tether-helm/dist/index.js` and `resources/tether-cli-hook/index.js` remain available as real files in the packaged app.

## ASAR, Fuses, And Signing

Current release posture:

- ASAR packaging is enabled.
- `node-pty` and `ssh2` are unpacked because they include native/runtime-loaded files.
- Helm MCP and CLI hook resources intentionally ship outside the ASAR because external CLIs spawn them by filesystem path.
- Electron fuses should disable `RunAsNode`, `NODE_OPTIONS`, and CLI inspect arguments.
- Cookie encryption, embedded ASAR integrity validation, and only-load-app-from-ASAR should stay enabled unless packaging smoke proves a breakage.
- `GrantFileProtocolExtraPrivileges` currently remains enabled because production windows are loaded from `file://`. Revisit this only with a custom app protocol migration.

Before public 1.0 GA, make an explicit signing decision:

- Windows: configure Authenticode signing for Squirrel artifacts, or mark the release as unsigned.
- macOS: configure Developer ID signing and notarization before publishing macOS artifacts.
- After packaging, verify fuse state with `@electron/fuses` against the packaged app binary.

## CSP And Renderer Sandbox

Current posture:

- `contextIsolation` is enabled.
- `nodeIntegration` is disabled.
- Main and docs windows have navigation/window-open guards.
- HTML CSP uses hashed inline boot scripts instead of broad inline script allowance.
- Inline styles remain allowed for the first-paint theme/bootstrap path.

Renderer `sandbox: true` is deferred for a dedicated migration because the main preload currently exposes `process.platform`, home directory discovery, clipboard, `webFrame`, and a broad IPC bridge. Treat sandbox enablement as a separate change with packaged-app smoke coverage across session creation, clipboard, zoom, docs, and settings flows.

## Release Blockers

Block 1.0 on:

- Any runtime audit finding.
- Any moderate, high, or critical root audit finding.
- Failing TypeScript, test, lint, Helm build, package, or make checks.
- Packaged app startup failure.
- Missing native module, Helm MCP, or CLI hook resources in the packaged app.
- An unresolved decision to publish unsigned public GA binaries.
